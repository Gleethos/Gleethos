Fovea - Fixed Foveated Rendering

This project represents my first encounter with the OpenGL API using C++.
I started the project as part of a computer graphics class in the 4th semester of my undergrad studies at the U.A.S Technikum.
Me and my team partner wanted to do something novel reflecting a current trend in computer graphics.

Therefore our choice fell on Foveated Rendering.
This rendering technique can get extremely complex when implementing it for full deployment, however we simply wanted to pull off a MVP for demonstration and learning how to program shader kernels.

We used a technique found in an interesting paper.
The concept we implemented takes the given vertecies in a scene and transforms them to a polar coordinate system which is then also modified by applying the log function to the radia of the vertices...
After rasterization we then simply apply the inverse (transform them back to the cartesian system) to the pixel coordinates and interpolate the pixels onto a new texture buffer.

Traditionally this technique goes hand in hand with eye tracking, however for obvious reasons we did not further explore such features.
